# 决策时限设定表

## 一，时限的重要性

### 1.1 为什么要设时限

```
没有时限的问题：
- 永远"再等等"
- 永远"再想想"
- 永远"还没准备好"
- 最后不了了之

设时限的价值：
- 创造紧迫感
- 逼迫决策
- 提高效率
- 打破拖延
```

### 1.2 时限类型

| 类型 | 时长 | 适用场景 |
|------|------|----------|
| 即时 | 2分钟内 | 小决策，如回复消息 |
| 快速 | 当天 | 简单决策，如选择A/B |
| 标准 | 1-3天 | 一般决策，如方案选择 |
| 重要 | 1周内 | 重要决策，如方向选择 |
| 重大 | 1月内 | 战略决策，如是否进入市场 |

## 二，时限设定原则

### 2.1 时限设定原则

```markdown
## 时限设定原则

### 原则1：越短越好
- 在保证质量的前提下
- 时限越短越好
- 避免无限期拖延

### 原则2：写下来
- 口头承诺不可靠
- 必须书面记录
- 公之于众更好

### 原则3：有惩罚
- 没有惩罚的时限形同虚设
- 设定自我惩罚
- 公开承诺增加压力

### 原则4：可调整
- 时限可以调整
- 但要说明原因
- 调整次数有限制
```

### 2.2 时限设定检查

```markdown
## 时限设定检查

□ 设定了明确的时间点
□ 设定了具体的日期
□ 时限足够短（越短越好）
□ 时限已经写下来
□ 有惩罚措施
□ 有人知道这个时限
□ 自己承诺按时完成
```

## 五，时限执行技巧

### 5.1 逼迫决策法

```markdown
## 逼迫决策法

### 方法1：设闹钟
- 为每个决策设闹钟
- 闹钟响必须做决定
- 不给自己找借口

### 方法2：公开承诺
- 告诉别人你的决策时限
- 让别人监督你
- 公开承诺增加压力

### 方法3：先开枪后瞄准
- 先做一个小决定
- 在行动中调整
- 避免完美主义

### 方法4：设置惩罚
- 没按时决策 = 惩罚
- 惩罚要足够痛
- 例如：做一件不想做的事
```

### 5.2 决策加速技巧

```markdown
## 决策加速技巧

### 技巧1：限制选项
- 不要给自己太多选择
- 最多3个选项
- 选项太多 = 拖延

### 技巧2：设定默认值
- 事先设定默认选项
- 不做决定 = 选默认值
- 例如：不做决定 = 选择当前方案

### 技巧3：使用硬币决策
- 把选择交给硬币
- 硬币落地后
- 感受你的反应
- 如果失望 = 你想要的是另一个

### 技巧4：倒计时决策
- 给自己5分钟
- 5分钟内必须决定
- 训练快速决策能力
```

## 七，常见问题与应对

### 7.1 时限设定问题

| 问题 | 原因 | 应对 |
|------|------|------|
| 时限太长 | 害怕做决定 | 缩短时限 |
| 时限太短 | 过于理想 | 留buffer |
| 时限无法遵守 | 优先级不对 | 重新排序 |
| 不断调整时限 | 没有约束 | 设硬性底线 |

### 7.2 决策拖延问题

| 问题 | 表现 | 应对 |
|------|------|------|
| 完美主义 | 永远觉得不够好 | 设定"够好"标准 |
| 害怕失败 | 担心选错 | 接受不完美 |
| 信息不足 | 总觉得不够了解 | 设信息截止期 |
| 选择困难 | 太多选项 | 减少选项 |
